# Частина 1

## Тема, Мета, Місце розташування

**Тема:** Розробка вебзастосунку інтернет-магазину квітів з адаптивним інтерфейсом.

**Мета:** Метою роботи є розробка вебзастосунку інтернет-магазину квітів, а також формування основних складових опису інформаційної системи, зокрема визначення актуальності, мети та завдань, об’єкта і предмета дослідження, опис бізнес-логіки, а також формування функціональних і нефункціональних вимог до системи.

## Актуальність теми

У сучасних умовах розвитку інформаційних технологій все більшої популярності набувають онлайн-сервіси для продажу товарів, зокрема інтернет-магазини. Користувачі надають перевагу швидкому та зручному доступу до продукції через вебінтерфейс, що дозволяє переглядати асортимент, порівнювати товари та здійснювати замовлення в будь-який час. Квіткові магазини також активно переходять у цифровий формат, оскільки це дає можливість розширити аудиторію та покращити якість обслуговування клієнтів.

Особливо важливим є створення адаптивного інтерфейсу, який забезпечує коректне відображення сайту на різних пристроях, таких як смартфони, планшети та персональні комп’ютери. Таким чином, розробка вебзастосунку інтернет-магазину квітів є актуальною задачею, що має практичне значення.

## Завдання роботи

Для досягнення поставленої мети необхідно виконати такі завдання:
- проаналізувати предметну область інтернет-магазину квітів;
- визначити основні складові інформаційної системи;
- сформулювати об’єкт і предмет роботи;
- описати бізнес-логіку роботи системи;
- сформувати функціональні вимоги до системи;
- сформувати нефункціональні вимоги;
- реалізувати базову структуру вебзастосунку з адаптивним інтерфейсом;
- підготувати матеріали для подальшого моделювання системи.

## Об’єкт та предмет роботи

Об’єктом роботи є вебзастосунок інтернет-магазину квітів, який призначений для представлення асортименту продукції та взаємодії з користувачами.

Предметом роботи є процеси розробки адаптивного користувацького інтерфейсу, організація структури вебзастосунку, а також визначення функціональних можливостей і вимог до інформаційної системи.

## Опис предметного середовища

Предметним середовищем даної роботи є діяльність інтернет-магазину квітів. Такий магазин надає користувачам можливість ознайомитися з асортиментом квіткової продукції, включаючи букети, композиції та декоративні рослини. Користувач може переглядати товари, ознайомлюватися з їх описом, цінами та обирати відповідні варіанти для покупки.

Система повинна забезпечувати зручну навігацію між різними категоріями товарів, такими як троянди, весільні букети, кімнатні рослини та інші. Окрім цього, важливими елементами є відображення інформації про доставку, контактні дані магазину та загальна інформація про компанію.

У подальшому система може бути розширена шляхом додавання функцій оформлення замовлення, обробки даних користувачів та інтеграції з серверною частиною.

## Бізнес-логіка системи

Бізнес-логіка системи визначає основні процеси взаємодії користувача з вебзастосунком. Користувач заходить на сайт інтернет-магазину, переглядає доступний асортимент квітів та обирає необхідні товари. Після цього він може ознайомитися з деталями товару, включаючи опис та ціну.

У разі оформлення замовлення система повинна зберігати інформацію про вибрані товари, дані користувача та загальну суму покупки. Далі замовлення передається на обробку, після чого здійснюється доставка.

З боку адміністратора передбачається можливість керування асортиментом товарів, оновлення інформації про продукцію, а також контроль замовлень.

## Функціональні вимоги

До основних функціональних вимог системи належать:
- відображення каталогу квіткової продукції;
- поділ товарів за категоріями;
- відображення інформації про товар (назва, опис, ціна, зображення);
- відображення спеціальних пропозицій;
- надання інформації про доставку;
- відображення контактної інформації;
- можливість подальшого розширення системи для оформлення замовлень.

## Нефункціональні вимоги

До нефункціональних вимог належать:
- адаптивність інтерфейсу для різних типів пристроїв;
- швидке завантаження сторінок;
- зручність та інтуїтивність користування;
- привабливий та сучасний дизайн;
- сумісність із сучасними браузерами;
- структурованість коду та можливість його подальшого розширення;
- стабільність роботи вебзастосунку.

## Use-case діаграма системи

![Use-case діаграма інтернет-магазину квітів](/assets/labs/lab-1/use-case.png)
 
На діаграмі показано основні сценарії взаємодії користувачів з системою інтернет-магазину квітів. У системі є два основні актори:
1. **Користувач** - може переглядати головну сторінку, каталог товарів, категорії товарів, інформацію про доставку, товар, а також оформлювати замовлення.
2. **Адміністратор** - може додавати, редагувати та видаляти товари, переглядати замовлення, змінювати їх статус.

Діаграма відображає основні функціональні можливості системи, зокрема, які дії можуть виконувати різні актори при взаємодії з інтерфейсом.

## ER-діаграма системи

![ER-діаграма інтернет-магазину квітів](/assets/labs/lab-1/er-diagram.png)

ER-діаграма відображає структуру бази даних інтернет-магазину квітів, де представлені основні сутності:
1. **Customer (Клієнт)** - зберігає інформацію про клієнта (ім'я, телефон, електронну пошту, адресу).
2. **Order (Замовлення)** - містить інформацію про замовлення (статус, сума, дата).
3. **Product (Товар)** - відображає товар (назва, опис, ціна, зображення).
4. **Category (Категорія)** - товар належить до певної категорії (наприклад, "Троянди", "Весільні букети").
5. **Order_Item (Позиція замовлення)** - містить зв’язок між товарами і замовленнями (які товари були замовлені, їх кількість і ціна).

Зв’язки між сутностями:
- Клієнт може оформити багато замовлень.
- Замовлення може містити кілька товарів.
- Кожен товар належить до певної категорії.
- Кожен товар має кілька позицій замовлень.

  ```js
  app.use(express.json());
  ```

### Завдання 4:

- Код для **PUT**:
  ```js
  app.put("/students/:id", (req, res) => {
    const { id } = req.params;
    const { name, group } = req.body;

    const student = students.find((s) => s.id == id);
    if (!student) {
      return res.status(404).json({ message: "Student not found" });
    }

    student.name = name;
    student.group = group;

    res.json({ message: "Student updated", student });
  });
  ```

- Код для **DELETE**:
  ```js
  app.delete("/students/:id", (req, res) => {
    const { id } = req.params;

    const index = students.findIndex((s) => s.id == id);
    if (index === -1) {
      return res.status(404).json({ message: "Student not found" });
    }

    students.splice(index, 1);
    res.json({ message: "Student deleted" });
  });
  ```
