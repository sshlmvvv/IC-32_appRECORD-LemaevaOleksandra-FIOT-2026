# Лабораторна робота №2: «СТВОРЕННЯ БАЗИ ДАНИХ У MYSQL. ПІДКЛЮЧЕННЯ NODE.JS ДО MYSQL. РОБОТА З ORM SEQUELIZE. »

## Мета роботи
- Навчитися створювати базу даних у MySQL
- Освоїти виконання SQL-запитів (SELECT, INSERT, UPDATE, DELETE)
- Підключати серверну програму на Node.js до бази даних
- Використовувати ORM Sequelize для роботи з БД
- Реалізувати зв’язок One-to-Many між таблицями

## 1. Підготовка середовища та встановлення залежностей
Для роботи серверної частини був ініціалізований проєкт Node.js та встановлені необхідні пакети:
- `express` — для створення веб-сервера.
- `mysql2` — драйвер для підключення до бази даних MySQL.
- `sequelize` — сучасна ORM для зручної роботи з БД через об'єкти JavaScript.

## 2. Проєктування бази даних (XAMPP)
За допомогою локального сервера XAMPP (Apache + MySQL) було створено базу даних `flower_shop_db`. 
Архітектура бази складається з двох пов'язаних таблиць:
1. **Categories** (Категорії квітів: Троянди, Тюльпани, Класика).
2. **Bouquets** (Конкретні букети з полями: назва, опис, ціна, посилання на фото та `CategoryId`).

## 3. Робота з чистим SQL (Частина 1)
На першому етапі робота з базою даних виконувалася через "чисті" SQL-запити (за допомогою пакета `mysql2`). Було створено скрипт `part1_mysql.js`, який дозволяє виконувати всі CRUD-операції напряму:
- **C:** Створення таблиць (`CREATE TABLE`) та додавання нових категорій/товарів (`INSERT INTO`).
- **R:** Пошук існуючих записів (`SELECT`).
- **U:** Оновлення даних, наприклад ціни (`UPDATE`).
- **D:** Видалення товару з бази (`DELETE`).

Оскільки цей скрипт підключений до тієї ж бази даних, що і головний сервер, будь-які зміни (наприклад, додавання нових Тюльпанів) миттєво відображаються на клієнтському сайті при його оновленні.

## 4. Перехід на ORM Sequelize (Частина 2)
Для більш сучасної та безпечної роботи був налаштований Sequelize.

## 6. Демонстрація CRUD-операцій
Для керування даними без зупинки основного сервера був створений окремий скрипт `sequelize.js`. Він демонструє повний цикл CRUD:
- **C (Create):** Створення категорії через `findOrCreate` та додавання нового букета "Величезна Корзина Троянд".
- **R (Read):** Пошук букета за назвою разом із його категорією (`include: Category`).
- **U (Update):** Зміна ціни знайденого букета.
- **D (Delete):** Видалення товару з бази.
