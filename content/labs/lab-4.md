## Мета роботи

- Набуття практичних навичок роботи з middleware для аналізу та обробки даних.

- Налаштування моніторингу продуктивності сервера та інтеграція менеджера процесів PM2 для фонової роботи.

### Завдання 2. Логування HTTP-запитів
Для відстеження всіх HTTP-запитів, які надходять до сервера, підключено middleware `morgan`. Це дозволяє автоматично виводити в консоль метод запиту, URL, статус-код та час виконання.

**Програмний код (`server.js`):**
```javascript

```

### Завдання 3. Файлове логування подій
Для надійного збереження історії роботи сервера інтегровано бібліотеку `winston`. Налаштовано запис логів у файл `app.log` з фіксацією рівнів `info` (інформаційні повідомлення) та `error` (помилки), а також додаванням точного часу (timestamp).

**Програмний код (`server.js`):**
```javascript
const winston = require("winston");

const logger = winston.createLogger({
    level: 'info',
    format: winston.format.combine(
        winston.format.timestamp(),
        winston.format.errors({ stack: true }),
        winston.format.json()
    ),
    transports: [
        new winston.transports.File({ filename: 'app.log' }),
        new winston.transports.Console({ format: winston.format.simple() }) 
    ]
});
```
![Скріншот: Вміст файлу app.log із записами подій](/assets/labs/lab-4/task3_winston.png)

### Завдання 4. Обробка помилок
Реалізовано глобальне middleware для обробки помилок. Воно перехоплює збої в роботі системи, записує детальну інформацію про помилку через `winston` у файл `app.log` та повертає користувачу коректну JSON-відповідь із кодом помилки, запобігаючи "падінню" сервера.

**Програмний код (`server.js`):**
```javascript
app.use((err, req, res, next) => {
    // Логування помилок через Winston
    logger.error(`Помилка: ${err.message}`); 
    
    // Повернення користувачу JSON-відповіді з кодом помилки
    res.status(err.status || 500).json({ 
        error: err.message || "Внутрішня помилка сервера" 
    });
});
```

### Завдання 5. Завантаження одного файлу
![Скріншот: Успішне завантаження](/assets/labs/lab-4/task5.png)

Реалізовано endpoint `/upload` з використанням бібліотеки `multer`. Цей маршрут дозволяє користувачеві (або флористу) завантажити один файл (ескіз чи фото букета), який автоматично зберігається на диску сервера.

**Програмний код (`server.js`):**
```javascript
const multer = require("multer");
const storage = multer.diskStorage({
    destination: (req, file, cb) => cb(null, 'uploads/'),
    filename: (req, file, cb) => cb(null, Date.now() + '-' + file.originalname)
});
const upload = multer({ storage: storage });

app.post('/upload', upload.single('file'), (req, res) => {
    if (!req.file) throw new Error("Файл не вибрано");
    logger.info(`Завантажено один файл: ${req.file.filename}`);
    res.json({ message: 'Файл завантажено успішно', file: req.file });
});
```

### Завдання 6. Завантаження кількох файлів
Функціонал було розширено для підтримки множинного завантаження. Створено endpoint `/upload-multiple`, який за допомогою методу `upload.array()` приймає одразу декілька файлів та зберігає їх у папку `/uploads`.

**Програмний код (`server.js`):**
```javascript
app.post('/upload-multiple', upload.array('files', 5), (req, res) => {
    if (!req.files || req.files.length === 0) throw new Error("Файли не вибрані");
    logger.info(`Завантажено декілька файлів: ${req.files.length} шт.`);
    res.json({ message: 'Файли завантажено успішно', files: req.files });
});
```

### Завдання 7. Валідація файлів
Для захисту сервера додано перевірки під час завантаження. У конфігурацію `multer` додано `fileFilter` для обмеження за типами (дозволено лише `.jpg`, `.png`, `.pdf`) та властивість `limits` для обмеження розміру файлу (до 5 МБ).

**Програмний код (`server.js`):**
```javascript
const fileFilter = (req, file, cb) => {
    const allowedTypes = ['image/jpeg', 'image/png', 'application/pdf'];
    if (allowedTypes.includes(file.mimetype)) {
        cb(null, true);
    } else {
        cb(new Error('Недопустимий тип файлу. Дозволено лише JPG, PNG та PDF.'), false);
    }
};

const uploadConfig = multer({ 
    storage: storage, 
    fileFilter: fileFilter, 
    limits: { fileSize: 5 * 1024 * 1024 } // Обмеження 5MB
});
```

### Завдання 8. Моніторинг стану сервера
Створено endpoint `/status`, який використовує вбудовані методи об'єкта `process` для надання даних про технічний стан застосунку.

**Програмний код (`server.js`):**
```javascript
app.get('/status', (req, res) => {
    res.json({
        uptime: process.uptime(), // Час безперервної роботи
        memoryUsage: process.memoryUsage(), // Статистика використання пам'яті
        timestamp: new Date()
    });
});
```

### Завдання 9. Вимірювання часу відповіді
Реалізовано спеціальне middleware, яке запам'ятовує час надходження запиту (`Date.now()`) і, використовуючи подію `finish` об'єкта `res`, розраховує тривалість його обробки. Результат записується в лог `winston`.

**Програмний код (`server.js`):**
```javascript
app.use((req, res, next) => {
    const start = Date.now();
    res.on('finish', () => {
        const duration = Date.now() - start;
        logger.info(`${req.method} ${req.url} - ${duration}ms`); 
    });
    next();
});
```

Окрім цього, я розробила власні middleware для вимірювання часу обробки запитів і централізованої обробки помилок (завдання 4, 9). Моніторинг ресурсів через `/status` та управління через PM2 (завдання 8, 10) забезпечили високу доступність сервера та захист від падінь. Отримані навички є фундаментом для створення відмовостійких та масштабованих веб-систем, готових до розгортання на реальних серверах.

## Контрольні питання

1. **Для чого потрібне логування в Node.js?**
   *Логування необхідне для пошуку помилок у коді (дебаггінгу), контролю дій користувачів, аналізу продуктивності (виявлення повільних запитів) та забезпечення безпеки системи шляхом фіксації підозрілої активності.*

2. **У чому різниця між Morgan та Winston?**
   *`Morgan` спеціалізується виключно на логуванні HTTP-запитів (автоматично записує метод, маршрут, статус-код, час). `Winston` - це універсальний і гнучкий логер, який дозволяє розробнику записувати будь-які системні події чи помилки, налаштовувати рівні важливості (info, error) та зберігати їх у файли або бази даних.*

3. **Як працює Multer та як реалізувати валідацію файлів?**
   *Multer - це middleware для обробки багаточастинкових даних (`multipart/form-data`), що перехоплює файл із запиту та зберігає його на диск. Валідація реалізується через функцію `fileFilter` (де перевіряється `mimetype` файлу) та об'єкт `limits` (де задається максимальний розмір файлу `fileSize`).*

4. **Які метрики повертає process.memoryUsage()?**
   *Вона повертає об'єкт із даними про використання оперативної пам'яті: `rss` (загальна пам'ять, виділена процесу), `heapTotal` (загальний обсяг пам'яті для об'єктів V8) та `heapUsed` (фактично використана пам'ять у даний момент).*

5. **Що таке PM2 та які його основні переваги?**
   *PM2 - це менеджер процесів для Node.js. Його основні переваги: запуск серверного застосунку у фоновому режимі, автоматичний рестарт при критичному падінні сервера, зручний моніторинг ресурсів системи та централізоване керування логами.*
